# Ozon ETL v2.0

## 📊 Описание

ETL-система для выгрузки данных из Ozon API в Google Таблицы. Собирает данные по:
- **Заказам** (FBS + FBO)
- **Аналитике** (воронка продаж: показы → клики → корзина → заказы)
- **Финансам** (продажи, комиссии, логистика, штрафы)
- **Остаткам** (из справочника)
- **Рекламе** (Performance API)

Результат — таблица из 26 столбцов с полной маржинальностью по каждому SKU.

---

## 🏗️ Архитектура

```
Ozon ETL v2.0/
├── 00_config.gs          # Конфигурация и константы
├── 01_core.gs            # Базовые классы (Cache, Logger, Secrets)
├── 02_clients.gs         # API клиенты (Ozon + Performance)
├── 03_etl.gs             # ETL процессы (загрузка → трансформация → запись)
└── 04_main.gs            # Точки входа, меню, UI
```

### Ключевые компоненты

| Компонент | Описание |
|-----------|----------|
| `OzonClient` | Клиент для Seller API Ozon с rate limiting и retry |
| `PerformanceClient` | Клиент для Performance API Ozon (реклама) |
| `CacheManager` | Кэширование через CacheService (TTL настраивается) |
| `SecretsManager` | Безопасное хранение токенов в PropertiesService |
| `LoggerManager` | Логирование в строку 3 таблицы (статус, прогресс) |

## 🚀 Быстрый старт

### 1. Установка

1. Скопируй все файлы в Google Apps Script проект
2. Настрой ключи через меню `📊 Ozon Отчеты → Установить учетные данные`

### 2. Создание листа

1. Нажми `📊 Ozon Отчеты → Создать лист для Ozon`
2. Введи название: `ozon_adv_effectiveness` (для первого кабинета)

### 3. Запуск

- `🔄 Магазин "JULE"` — для первого кабинета
- `🔄 "Ювелир Карат"` — для второго
- `🔄 Обновить все` — для обоих

## 🔑 Настройка ключей

### Seller API (обязательно)
1. Личный кабинет Ozon → Настройки → API-ключи
2. Создать сервисный аккаунт → Добавить ключ
3. Скопировать **Client-ID** и **API-Key**

### Performance API (для рекламы)
1. Личный кабинет Ozon → Настройки → API-ключи
2. Создать сервисный аккаунт для Performance API
3. Скопировать **Client ID** и **Client Secret**

## 📈 Оптимизация производительности

- **Кэширование** — все тяжелые запросы кэшируются на 1-24 часа
- **Rate limiting** — не более 50 запросов/сек (Seller API)
- **Пагинация** — данные загружаются постранично с задержками
- **Прогресс-бар** — статус отображается в строке 3 таблицы

---

## ⚠️ Ограничения

- Время выполнения — максимум 6 минут (Google Apps Script)
- Кэш — до 100 KB на ключ
- Реклама — требуется отдельная авторизация Performance API
